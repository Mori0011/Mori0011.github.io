这里是我的第一篇博客，接下来我将详细阐释一下blog的学习过程与心得

一、环境搭建

环境准备

  • 操作系统：Windows10
  • Node
  • Git
  • Hexo
  • 一个 GitHub 帐号
  • 一个云服务器（可选）
  • 一个域名（可选）

安装Node.js和Hexo

Node的下载链接在这里，直接傻瓜式安装就好啦：Node.js

安装好Node后即可使用以下命令进行安装Hexo

可以键入hexo -v验证是否成功安装

创建Github仓库

点击右上角的+按钮，选择New repository，创建一个<用户名>.github.io的仓库。

创建入口：

  • 仓库的格式必须为：<用户名>.github.io

  • Description：为描述仓库（选填）

  • 勾选 Initialize this repository with a README 初始化一个 README.md 文件

  • 点击 Creat repository 进行创建

  • Owner中的内容和Repostory name第一部分内容必须一致

  • 勾选Add a READNAME file选项

二、新建blog

初始化项目

会在当前目录下新建一个项目名的文件夹，并初始化相应内容

进入到blog的根目录下，键入npm i初始化依赖

初始化项目后，会出现如下tree

初始化项目后，blog-demo有如下结构：

  • 【node_modules】：依赖包
  • 【scaffolds】：生成文章的一些模板
  • 【source】：用来存放你的文章
  • 【themes】：主题
  • 【.npmignore】：发布时忽略的文件（可忽略）
  • 【_config.landscape.yml】：主题的配置文件
  • 【_config.yml】：博客的配置文件
  • 【package.json】：项目名称、描述、版本、运行和开发等信息https://github.com/jerryc127/hexo-theme-butterfly)

新建一篇自己的博客

博客都会存放于/source/_posts/目录下

输入hexo server或者hexo s后，即可看到博客的初始化界面，当然，原皮并不好看。这里强烈安利[butterfly主题](

挂载到Github上

安装 hexo-deployer-git

修改 _config.yml 文件

在blog-demo目录下的_config.yml，就是整个Hexo框架的配置文件了。可以在里面修改大部分的配置。详细可参考官方的配置描述。

修改最后一行的配置，将repository修改为你自己的github项目地址即可。

这里要注意一点，自己仓库的默认分支是否是master

部署项目到 GitHub

修改好配置后，运行如下命令，将代码部署到 GitHub。

  • hexo clean：删除之前生成的文件，若未生成过静态文件，可忽略此命令。
  • hexo generate：生成静态文章，可以用hexo g缩写
  • hexo deploy：部署文章，可以用hexo d缩写

如果出现Deploy done，则说明部署成功啦。
