前言

昨天把考纲框架搭好了，今天正式开始啃第一章——计算机系统基础和操作系统。说实话这块内容上学的时候学过，但毕业后这几年全还给老师了，基本上只剩个印象。

翻了翻教材，发现这章考的倒不算特别深，但有几个计算题特别容易出错：数制转换、补码计算、PV操作。安全服务干久了反而对这些底层原理有点生疏，趁这个机会好好补补。

📚 参考资料：本篇内容结合2026年软考网络工程师考试大纲[1]、希赛网考纲整理[2]、信管网历年真题[3]等权威资料进行系统性扩充学习。

1.4 存储器层次结构

从快到慢排列：

★ 关键知识点：

  1. Cache命中率：访问Cache的比例，越高越好

    • 命中率 = Cache命中次数 / 总访问次数

  2. 局部性原理：

    • 时间局部性：最近访问的地址近期还会访问
    • 空间局部性：访问某个地址后，附近的地址也可能被访问

  3. 主存容量计算：

    • 地址总线宽度为n位，则最大寻址空间 = 2^n 字节

💡 记忆技巧：越快的东西越贵、容量越小。这跟吃饭一样——CPU寄存器就像面前的小碟子，Cache是桌上的一盘菜，内存是厨房里的冰箱，磁盘是仓库里的货柜。

二、操作系统

2.1 进程状态转换

根据2026年软考网络工程师考纲，进程与线程、进程管理（含进程状态、转换）属于直接考查范围[1][2]。

进程三态模型：

★ 状态说明：

  • 运行态：进程正在CPU上执行
  • 就绪态：进程已具备运行条件，等待CPU调度
  • 阻塞态：进程因等待I/O或其他资源而暂停

📝 五态模型补充：在更复杂系统中，还有新建态和终止态，构成进程生命周期完整闭环。

2.2 PV操作（最头疼的部分）

PV操作是进程同步与互斥的核心机制，也是考试必考的重难点。根据考纲，进程间通信（IPC）与同步机制（信号量、消息、管道等）属于直接相关考点[1][2][3]。

信号量机制详解

★ 核心语义记忆（根据牛客网等权威解析[4]）：

操作 执行动作 阻塞条件 唤醒条件
P S = S - 1 S < 0 -
V S = S + 1 - S ≤ 0

💡 记忆技巧：P的首字母代表”减”（Pause/Proberen），V的首字母代表”加”（Verhogen）。P操作可能阻塞，V操作可能唤醒。

信号量分类

类型 初值 用途 示例
二值信号量 1 互斥 保护临界区
计数信号量 N 资源控制 打印机池、空闲缓冲区

解题步骤（个人总结）

  1. 分析题目中有几个进程、几个资源
  2. 确定信号量初值（互斥信号量=1，同步信号量=资源数）
  3. 画出各进程的代码框
  4. 按时间顺序分析P、V操作

2.3 死锁问题

根据考纲，死锁原理与避免、资源竞争与互斥属于直接相关考点[1][2]。

死锁四个必要条件

  1. 互斥条件：资源一次只能被一个进程使用
  2. 请求与保持条件：进程保持已有资源的同时，请求新资源
  3. 不可抢占条件：已分配资源不能被强制抢占
  4. 循环等待条件：形成进程-资源的循环链

★ 高频考点：破坏这四个条件中的任意一个，就能避免死锁。

2.4 作业调度算法

先来先服务（FCFS）

按到达时间先后顺序调度，简单但平均等待时间长。

短作业优先（SJF）

优先调度运行时间最短的作业。可能导致长作业”饿死”。

高响应比优先

响应比 = (等待时间 + 运行时间) / 运行时间

兼顾了等待时间和服务时间，避免了长作业饿死的问题。

★ 高频考点：计算各种调度算法的平均等待时间，这是下午题常考的计算题类型。

调度算法对比

算法 优点 缺点 是否抢占
FCFS 简单、公平 平均等待时间长 非
SJF 最短平均等待 长作业饿死 可非
高响应比 兼顾长短作业 计算开销大 非
时间片轮转 响应快、公平 时间片大小难定 是
优先级 灵活 低优先级饿死 可非

2.5 磁盘调度算法

先来先服务（FCFS）

按请求顺序依次处理，最简单但效率低。

最短寻道时间优先（SSTF）

优先处理距离当前磁头位置最近的请求。可能产生”饥饿”现象。

扫描算法（SCAN）

磁头从一端向另一端移动，遇到请求就处理，到达端后反向移动。又称”电梯算法”。

循环扫描算法（CSCAN）

磁头只能单方向移动，到达端后快速返回起点重新开始。

LOOK与C-LOOK

SCAN和CSCAN的优化版本，磁头到达最远请求即返回，不必须到达边界。

★ 高频考点：计算不同调度算法的平均寻道距离。

三、易错/踩坑记录

3.1 补码计算总是错

错题：求 -37 的8位二进制补码

我第一次做的时候：

  1. 37的二进制 = 00100101
  2. -37的原码 = 10100101
  3. -37的反码 = 11011010
  4. -37的补码 = 11011011 ✓

看起来对了，但题目问的是”用补码表示-37”，答案是 11011011。

❌ 踩坑原因：补码计算时符号位也要参与运算！容易在取反的时候忘记符号位的变化。

3.3 寻址方式判断失误

错题：判断指令 MOV A, #30H 是什么寻址方式

我选了”直接寻址”，，但正确答案是”立即寻址”。

❌ 踩坑原因：#30H 前面有井号，表示这是一个立即数常量，不是内存地址！

📝 纠错笔记：

  • MOV A, #30H → 立即寻址（操作数是常量）
  • MOV A, 30H → 直接寻址（30H是内存地址）
  • MOV A, @R0 → 寄存器间接寻址

3.4 PV操作阻塞条件混淆

❌ 错题：P操作后，当S<0时进程应该…？

我选了”继续执行”，正确答案是”阻塞进入等待队列”。

📝 纠错笔记（根据牛客网权威解析[4]）：

信号量值 P操作后 V操作后
S > 0 S减1，进程继续 S加1，进程继续
S = 0 S减1变负，进程阻塞 S加1变0，唤醒等待进程
S < 0 S继续减，进程阻塞 S加1，若仍≤0则唤醒

  • 地址空间布局随机化（ASLR）

4.2 权限控制与访问控制

操作系统的权限管理机制和等保测评中的访问控制要求高度一致：

操作系统概念 等保要求
用户/组 身份鉴别
文件权限（rwx） 自主访问控制
ACL访问控制列表 扩展访问控制
sudo提权 特权用户管理

做等保测评的时候，经常要检查服务器的权限配置是否遵循最小权限原则。

4.3 进程监控与应急响应

做应急响应的时候，经常要分析可疑进程。理解进程状态转换有助于判断系统是否异常：

  • 异常大量就绪进程 → 可能存在资源竞争或DDoS攻击
  • 进程长时间处于阻塞态 → 可能存在I/O异常或外部依赖问题
  • 僵尸进程 → 父进程未正确回收子进程，可能是攻击痕迹

📌 工作感悟：以前做渗透测试只知道工具怎么用，不懂底层原理。现在学了操作系统，才发现漏洞利用和防护都是基于这些基础原理。难怪面高级岗都要问操作系统内核机制。
