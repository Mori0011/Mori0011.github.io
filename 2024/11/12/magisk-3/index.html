初识smali，vip终结者

什么是JVM、Dalvik、ART

JVM时JAVA虚拟机，运行JAVA字节码程序

Dalvik时Google专门为Android设计的一个虚拟机，Dalvik有专属的文件执行格式dex（Dalvik executable）

Art（Android Runtime）相当于Dalvik的升级版，本质与Dalvik无异

smali及其语法

smali时Dalvik的寄存器语言，smali代码时dex反编译来的

下列是相关关键字及其信息

名称 注释
.class 类名
.super 父类名，继承的上级类名名称
.source 源名
.field 变量
.method 方法名
.register 寄存器
.end method 方法名的结束
public 公有
proteted 半公开，只有同一家人才能用
private 私有，只能自己使用
.parameter 方法参数
.prologue 方法开始
.line xxx 位于第xxx行

数据类型及其对应

smali类型 java类型 注释
V void 无返回值
Z boolean 布尔值类型，返回0或1
B byte 字节类型，返回字节
S short 短整数类型，返回数字
C char 字符类型，返回字符
I int 整数类型，返回整数
J long（64位 需要两个寄存器存储） 长整数类型，返回数字
F float 单浮点类型，返回数字
D double（64位 需要两个寄存器存储） 双浮点类型，返回数字
sting String 文本类型，返回字符串穿
Lxxx/xxx/xxx object 对象类型，返回对象

常用指令

关键字 注释
const 重写整数属性，真假属性内容，只能是数字类型
onst-string 重写字符串内容
const-wide 重写长整数类型，多用于修改到期时间
return 返回指令
if-eq 全称equal(a=b)，比较寄存器ab的内容，相同则跳转
if-ne 全称not equal(a!=b)，a和b的内容不相同则跳转
if-eqz 全称equalzero(a=0)，z即时0的标记，a等一0则跳转
if-nez 全称garden equal(a!=0)，a不等于0则跳转
if-ge 全称garden equal(a>=b)，a大于或等于则跳转
if-le 全称little equal(a<=b)，a小于或等于b则跳转
goto 强制跳转到指定位置
switch 分支跳转，一般会有多个分支线，并根据指令跳转到适当位置
iget 获取寄存器数据

其余指令可以使用语法工具查询

定位方法：搜索弹簧关键字、住区按钮id

修改方法：修改判断、强制跳转、修改寄存器的值

寄存器

在smali里的所有操作都必须经过寄存器来进行：本地寄存器用v开头数字结尾的符号来表示，例如v0、v1、v2。参数寄存器则使用p开头数字结尾的符号来表示，如p0、p1、p2、。特别值得注意的是，p0不一定是函数中的第一个桉树，在非static函数中，p0代指“this”，p1表示函数的第一个参数，p2代表函数中的第二个参数。而在static函数中p0才对应第一个参数（因为Java的static方法中没有this方法）
